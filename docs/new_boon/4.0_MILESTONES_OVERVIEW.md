# Chapter 4: Milestones

This chapter tracks major milestones for the Boon project, from the first production-ready playground to running Boon on custom hardware and beyond.

---

## Milestone Summary

| # | Title | Summary | Prerequisites |
|---|-------|---------|---------------|
| 0 | Boon CLI + Testing | Standalone CLI runtime with `boon test`, platform abstraction | - |
| 1 | New Engine | Playground works with new arena-based engine (shared with CLI) | M0 |
| 2 | Language Server | Inline types in editor, WebWorker LSP, type checker | M1 |
| 3 | FPGA Transpiler | Boon → SystemVerilog, digitaljs fork, super-counter on FPGA | M1 |
| 4 | Super-Counter | Distributed app + basic boon-console: button→Pi→clients | M0, M3 |
| 5 | Multi-File Modules | todo_mvc_physical + RayBox, virtual FS, BUILD.bn/RUN.bn | M1, RayBox |
| 6 | RISC-V Design | RISC-V processor in Boon (discovers synthesizable subset) | M3 |
| 7 | RISC-V Runtime | Boon runs in WAMR on RISC-V, FPGA state persistence (MRAM) | M6 |
| 8 | kavik.cz | Physical rendering, accessibility layer, SEO | M5, RayBox |
| 9 | Testing (Cross-Platform) | Browser/FPGA testing, Boon-native TEST {} syntax | M0, M1 |
| 10 | boon-console (Full) | 3D visualization, time-travel, state inspection | M4 |
| 11 | Servo-RayBox App | Desktop app with 3D visualizations from boon-console | M5, M10 |
| 12 | Performance | Profiling, benchmarks, optimization across all platforms | M11 |

## Milestone Progression

### Phase 0: Foundation (M0)
Create standalone CLI for rapid development iteration.

- **M0**: Boon CLI with `boon test`, platform abstraction traits

### Phase 1: Core Platform (M1-M2)
Establish the new reactive engine and developer tooling.

- **M1**: Migrate playground to arena-based engine (shared with CLI)
- **M2**: Add inline types and type checking in editor

### Phase 2: Hardware Bridge (M3-M4)
Bridge software and hardware with FPGA synthesis.

- **M3**: Transpile Boon to SystemVerilog
- **M4**: Demonstrate full super-counter distributed app with basic boon-console

### Phase 3: Rich Applications (M5)
Enable complex multi-file applications with 3D UI.

- **M5**: Virtual filesystem, modules, BUILD.bn/RUN.bn, RayBox integration

### Phase 4: Custom Processor (M6-M7)
Design and run Boon on custom RISC-V hardware.

- **M6**: Implement RISC-V processor in Boon (discover synthesizable subset)
- **M7**: Run Boon inside WAMR on RISC-V with persistent state

### Phase 5: Production Website (M8)
Real-world application with full accessibility and SEO.

- **M8**: kavik.cz rewritten in Boon with physical rendering

### Phase 6: Developer Experience (M9-M11)
Complete development and debugging infrastructure.

- **M9**: Cross-platform testing (browser, FPGA)
- **M10**: Full boon-console runtime diagnostics
- **M11**: Boon-Servo-RayBox desktop development environment

### Phase 7: Polish (M12)
Final optimization pass.

- **M12**: Performance profiling and optimization
